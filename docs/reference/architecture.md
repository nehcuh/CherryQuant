### 模拟账户 vs 实盘执行（教学设计说明）

在 CherryQuant 中，**策略 Agent 内部的模拟账户** 与 **通过 VNPy/CTP 的实盘执行** 是两条刻意区分的路径：

- 模拟账户（StrategyAgent.positions / trades）：
  - 在 `_execute_buy` / `_execute_sell` 中，一旦本地资金与风险检查通过，即视为“理想条件下完全成交”；
  - 账户价值、回撤、胜率等风险指标全部基于这一套“理想成交”账本计算；
  - 适合作为课堂上的 **策略逻辑与风控推演** 的主要载体。
- 实盘执行（KLineOrderManager.executions / AgentManager.live_executions）：
  - 真实订单通过 `KLineOrderManager → VNPyGateway → vn.py/CTP` 发送到交易所；
  - 成交回报从 vn.py 事件回调进入 `KLineOrderManager._on_trade_update`，再经由异步回调进入 `AgentManager._on_execution_update`；
  - 在 `live_executions` 中按策略聚合，供 Web 前端和 Notebook 生成 **Live PnL 曲线**。

> 重要：当前教学版本中，**实盘成交不会反向改写模拟账户的持仓与 PnL**，二者可能长期偏离。
> 这样设计是为了让学生清晰对比“模型预期 vs 实际执行”，而不是直接隐藏所有差异。

#### PnL 计算的简化假设

- 模拟 PnL：
  - 由 StrategyAgent 在每次卖出/平仓时按 `(平仓价 - 开仓价) * 手数 * 杠杆` 计算，并在 Notebook / API 中简单累加；
  - 忽略合约乘数、手续费细节和滑点，只用于展示曲线形状与方向。
- 实盘 PnL：
  - API 侧根据 `live_executions` 中的 `symbol / direction / volume / price` 进行简化撮合：
    - 为每个品种维护头寸与平均成本价；
    - 反向成交优先用于平仓，产生已实现 PnL；
    - 超出部分视为反向开仓；
  - 同样忽略合约乘数和复杂费用，仅用作近似对比曲线。

在生产级系统中，这两条路径通常会通过“对账与回写”机制合并；而在 CherryQuant 的教学版中，我们保留这层差异，用以说明策略实现、订单执行和实际收益之间的多层关系。

                                                ↓

## 📈 后续优化方向与高级课题（课程预留）

> 本节面向任课教师和进阶学生，记录 **下一阶段可以纳入课程的优化方向**，当前代码库不一定全部实现，
> 可作为毕业项目或高级 Lab 的题目池。

### 1. 对账与状态回写（Reconciliation）

- 在当前版本基础上，设计并实现「软对账 / 硬对账」两种模式：
  - 软对账：仅基于 `live_executions` 生成 Live PnL 曲线，不回写账户状态（当前实现）。
  - 硬对账：以实盘成交为准重建持仓和现金流，并与模拟账户做差异对比。
- 增加组合级 `reporting_mode`：`"sim" | "live" | "mixed"`，在 Web / 报表中切换不同口径。
- 研究跨日持仓、部分成交、手续费、资金占用等因素对对账的影响。

### 2. 多账户与分账户风控

- 将当前单账户模型扩展为「母账户 + 多子账户」结构：
  - 不同策略或策略组映射到不同子账户；
  - 在组合层做风险预算与额度分配。
- 设计课程实验，引导学生：
  - 对比单账户与多账户架构的优劣；
  - 思考不同维度（策略 / 品种 / 团队）的风险隔离。

### 3. 延迟、滑点与交易成本建模

- 在模拟账户侧加入可配置的：
  - 撮合延迟分布（固定 / 正态 / 尾部加重）；
  - 滑点模型（按价差、按委托量、按盘口深度）；
  - 手续费与保证金模型（按交易所规则近似）。
- 设计对比实验：
  - 同一策略在「零成本 / 理想成交」与「现实成交」假设下的 PnL 差异；
  - 学生需要给出“模型收益”向“可实现收益”折扣的合理范围。

### 4. 异步架构与稳健性

- 基于当前 asyncio + vn.py 集成，进一步探讨：
  - 如何设计统一的任务调度与取消机制；
  - 如何在高负载下做 backpressure 与降级处理；
  - 如何监控事件循环卡顿与长尾任务。
- 可结合 `pytest-asyncio` 或自定义工具，增加「异步稳定性测试」Lab。

### 5. AI 决策与策略生成的高级主题

- 探索多模型协同决策：
  - 例如 GPT-4 负责逻辑推理，GLM-4 负责本地化信息与中文解释；
  - 引导学生设计简单的「模型路由 / 加权投票」机制。
- 尝试在教学环境中引入：
  - 经验回放 + 策略描述的 few-shot 训练；
  - 对历史决策数据的离线评估与再提示（Re-prompting）。

> 建议在第一轮课程中 **保持当前简化实现不变**，待学生熟悉主干流程后，
> 再选择其中 1-2 个方向作为新一轮课程或毕业项目的主题。
